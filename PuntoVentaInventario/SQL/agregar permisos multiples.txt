Sí, se puede dejar **totalmente reutilizable** para otras vistas. Angular permite aplicar guards a múltiples rutas y también pasar datos personalizados por ruta, así que en vez de hacer algo “solo para `venta/venta`”, puedes convertirlo en un sistema de PIN reutilizable por pantalla o por grupo de pantallas. [angular](https://angular.dev/guide/routing/route-guards)

## La forma correcta de reutilizarlo

Lo mejor es que el guard no esté amarrado a una sola ruta fija, sino que lea información desde `data` de cada ruta, por ejemplo un `pinKey` o un `scope`. Angular soporta metadata en rutas y los guards pueden usarla para decidir si dejan pasar o no. [mintlify](https://mintlify.wiki/angular/angular/routing/route-guards)

Por ejemplo:
- `venta/venta` pide PIN de ventas.
- `configuracion/usuarios` pide PIN de admin.
- `reportes/cierre-caja` pide PIN de caja. [coreui](https://coreui.io/answers/how-to-create-route-guard-in-angular/)

## Cómo lo haría reusable

En vez de tener una sola bandera `accesoVistaProtegida = false`, guarda accesos por clave. Así el servicio puede manejar varias vistas protegidas sin duplicar lógica. Un `Map<string, boolean>` encaja perfecto para eso. [angular](https://angular.dev/guide/routing/route-guards)

### `pin-access.service.ts`
```ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';
import { environment } from '../../environments/environment.development';

export interface ValidarPinResponseDto {
  autorizado: boolean;
  mensaje: string;
}

@Injectable({
  providedIn: 'root'
})
export class PinAccessService {

  private readonly urlBase = environment.apiURL + '/Seguridad';
  private accesos = new Map<string, boolean>();

  constructor(private http: HttpClient) {}

  validarPin(pin: string, scope: string): Observable<ValidarPinResponseDto> {
    return this.http.post<ValidarPinResponseDto>(`${this.urlBase}/validar-pin`, { pin, scope });
  }

  concederAcceso(scope: string): void {
    this.accesos.set(scope, true);
  }

  revocarAcceso(scope: string): void {
    this.accesos.delete(scope);
  }

  revocarTodos(): void {
    this.accesos.clear();
  }

  tieneAcceso(scope: string): boolean {
    return this.accesos.get(scope) === true;
  }
}
```

## Guard reusable

El guard puede leer `scope` desde `route.data`. Eso hace que el mismo guard sirva para muchas rutas distintas. Angular route guards pueden leer `data` de la ruta actual. [stackoverflow](https://stackoverflow.com/questions/49800298/custom-guard-get-id-route-params)

### `pin.guard.ts`
```ts
import { inject } from '@angular/core';
import { ActivatedRouteSnapshot, CanActivateFn, Router } from '@angular/router';
import { PinAccessService } from '../services/pin-access.service';

export const pinGuard: CanActivateFn = (route: ActivatedRouteSnapshot) => {
  const pinAccessService = inject(PinAccessService);
  const router = inject(Router);

  const scope = route.data['pinScope'] as string;

  if (!scope) {
    return router.parseUrl('/home');
  }

  if (pinAccessService.tieneAcceso(scope)) {
    return true;
  }

  return router.parseUrl('/home');
};
```

## Rutas

Ahora puedes usar el mismo guard en muchas rutas y solo cambiar el `pinScope`. Angular permite exactamente este patrón con `data` en configuración de rutas. [blog.nashtechglobal](https://blog.nashtechglobal.com/implementing-nested-routes-and-route-guard-in-angular/)

### `app.routes.ts`
```ts
export const routes: Routes = [
  { path: 'home', component: HomeComponent },

  { path: 'venta/realizar-venta', component: RealizarVentaComponent },
  {
    path: 'venta/venta',
    component: VentaComponent,
    canActivate: [pinGuard],
    data: { pinScope: 'ventas' }
  },

  {
    path: 'admin/usuarios',
    component: UsuariosAdminComponent,
    canActivate: [pinGuard],
    data: { pinScope: 'admin-usuarios' }
  },

  {
    path: 'reportes/cierre-caja',
    component: CierreCajaComponent,
    canActivate: [pinGuard],
    data: { pinScope: 'cierre-caja' }
  },

  { path: '', redirectTo: 'home', pathMatch: 'full' },
  { path: '**', redirectTo: 'home' }
];
```

## Backend para múltiples vistas

Si quieres reutilizarlo bien también del lado servidor, el backend puede recibir `scope` y comparar contra distintos PINs del `appsettings.json`. ASP.NET Core lee secciones de configuración fácilmente, así que puedes guardar varios pins por nombre. [stackoverflow](https://stackoverflow.com/questions/46940710/getting-value-from-appsettings-json-in-net-core)

### `appsettings.json`
```json
{
  "Security": {
    "Pins": {
      "ventas": "230815",
      "admin-usuarios": "998877",
      "cierre-caja": "445566"
    }
  }
}
```

### DTO request
```csharp
public class ValidarPinRequestDto
{
    public string Pin { get; set; } = string.Empty;
    public string Scope { get; set; } = string.Empty;
}
```

### Controller
```csharp
[HttpPost("validar-pin")]
public ActionResult<ValidarPinResponseDto> ValidarPin([FromBody] ValidarPinRequestDto dto)
{
    try
    {
        if (dto == null || string.IsNullOrWhiteSpace(dto.Pin) || string.IsNullOrWhiteSpace(dto.Scope))
        {
            return BadRequest(new ValidarPinResponseDto
            {
                Autorizado = false,
                Mensaje = "El pin y el scope son obligatorios."
            });
        }

        var pinConfigurado = _configuration[$"Security:Pins:{dto.Scope}"];

        if (string.IsNullOrWhiteSpace(pinConfigurado))
        {
            return NotFound(new ValidarPinResponseDto
            {
                Autorizado = false,
                Mensaje = "No existe configuración para esta vista protegida."
            });
        }

        if (dto.Pin.Trim() != pinConfigurado.Trim())
        {
            return Unauthorized(new ValidarPinResponseDto
            {
                Autorizado = false,
                Mensaje = "Pin incorrecto."
            });
        }

        return Ok(new ValidarPinResponseDto
        {
            Autorizado = true,
            Mensaje = "Acceso concedido."
        });
    }
    catch (Exception ex)
    {
        return StatusCode(500, new ValidarPinResponseDto
        {
            Autorizado = false,
            Mensaje = $"Error al validar el pin: {ex.Message}"
        });
    }
}
```

## Navbar reusable

También puedes abrir el mismo modal para cualquier vista, solo cambiando ruta y scope antes de mostrarlo. Así no duplicas componentes ni lógica. [syncfusion](https://www.syncfusion.com/blogs/post/implementing-route-protection-in-angular-using-canactivate)

### Idea en `NabvarComponent`
```ts
mostrarModalPin = false;
rutaProtegidaPendiente = '';
scopePendiente = '';

abrirVistaProtegida(ruta: string, scope: string): void {
  if (this.pinAccessService.tieneAcceso(scope)) {
    this.router.navigate([ruta]);
    return;
  }

  this.rutaProtegidaPendiente = ruta;
  this.scopePendiente = scope;
  this.mostrarModalPin = true;
}

confirmarPin(pin: string): void {
  this.pinAccessService.validarPin(pin, this.scopePendiente).subscribe({
    next: (response) => {
      if (response.autorizado) {
        this.pinAccessService.concederAcceso(this.scopePendiente);
        this.mostrarModalPin = false;
        this.router.navigate([this.rutaProtegidaPendiente]);
      }
    },
    error: () => {
      this.toastrService.error('Pin incorrecto.');
    }
  });
}
```

</button>

</button>
